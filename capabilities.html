PWA Capabilities

A clear overview of what Progressive Web Apps can — and cannot — do today.

✅ What a PWA Can Do

📲

Install to Home Screen

Users can add the PWA to their device's home screen, giving it an app-like icon and launch experience without visiting an app store.

🔌

Work Offline

Service workers intercept network requests and serve cached assets, letting the app function even without an internet connection.

🔔

Push Notifications

With user permission, PWAs can receive push notifications from a server — keeping users informed even when the app is not open.

📍

Geolocation

Access the device's GPS or network-based location (with user permission) to provide location-aware features.

📷

Camera & Microphone

Use the device camera and microphone via the MediaDevices API (with user permission) for photo capture, video calls, and voice input.

🔄

Background Sync

Queue actions (e.g., form submissions) while offline and automatically replay them once connectivity is restored.

📂

File System Access

The File System Access API allows reading and writing files on the user's device (with explicit permission), enabling desktop-class editing experiences.

🔗

Web Share

Invoke the native OS share sheet to share URLs, text, or files to other apps installed on the device.

⚡

Fast Loading

Pre-cached assets load instantly on repeat visits, offering performance that rivals — or exceeds — native apps.

❌ What a PWA Cannot Do

📦

App Store Distribution

PWAs are not natively listed in the Apple App Store or Google Play Store without additional packaging (e.g., Trusted Web Activity / Capacitor).

🔒

Full iOS Support

Safari on iOS has historically lagged behind: background sync, Web Bluetooth, and some notification features are absent or limited on iOS devices.

📡

Bluetooth & NFC

Web Bluetooth and Web NFC APIs are not universally supported. Safari does not support them, limiting their real-world applicability.

📞

Contacts & SMS Access

PWAs cannot read the device address book or send / receive SMS messages, unlike native apps on Android and iOS.

🔧

Deep System Integration

Features such as modifying system settings, running persistent background services, or inter-app communication are not available to PWAs.

🖥️

Cross-Origin Data Access

The browser's same-origin policy restricts PWAs from reading data from other origins without explicit CORS headers from those servers.
